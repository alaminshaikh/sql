Purpose of ORDER BY and HAVING Clauses in SQL

i) ORDER BY Clause

The ORDER BY clause is used to sort the result set of a query by one or more columns. The sorting can be in ascending (ASC) or descending (DESC) order. By default, the sorting is in ascending order if no order is specified.

Purpose

  • Sorting Results: The primary purpose is to organize the query results in a specific order.
  • Enhancing Readability: Makes it easier to read and analyze the results, especially when dealing with large datasets.

Syntax

SELECT column1, column2, ...
FROM table_name
ORDER BY column1 [ASC|DESC], column2 [ASC|DESC], ...;

Example

SELECT first_name, last_name, age
FROM students
ORDER BY last_name ASC, age DESC;

In this example, the results are sorted first by last_name in ascending order, and then by age in descending order within each last_name.

ii) HAVING Clause

The HAVING clause is used to filter records that have been aggregated by the GROUP BY clause. It is similar to the WHERE clause, but HAVING is used to filter groups rather than individual rows.

Purpose

  • Filtering Groups: It allows for conditions to be applied to groups of rows created by the GROUP BY clause.
  • Aggregate Functions: It is typically used with aggregate functions (e.g., SUM, COUNT, AVG) to filter groups based on aggregate values.

Syntax

SELECT column1, aggregate_function(column2)
FROM table_name
GROUP BY column1
HAVING condition;

Example

SELECT department, COUNT(employee_id) AS num_employees
FROM employees
GROUP BY department
HAVING COUNT(employee_id) > 10;

In this example, the query retrieves the departments that have more than 10 employees. The HAVING clause filters the groups formed by GROUP BY based on the aggregate count of employee_id.
